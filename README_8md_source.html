1 JD-sample-libraries
2 ===================
3 
4 Dirt-simple sample libraries for EECE 387. These libraries are intended to be minimalist samples-- they don't take advantage of more complex hardware features such as interrupt synchronization.
5 
6 
7 Libraries
8 -----------
9 
10 - A <i>Two Wire Interface ("I2C") master</i> library, which allows simple control of an I2C device using a bus-pirate-like syntax. See <a href="http://ktemkin.github.io/JD-sample-libraries/master_8h.html">the documentation for <code>twi/master.h</code></a>, or the samples below.
11 - A <i>uart-over-stdio</i> library, which is conveneint for simple serial monitors. See <a href="http://ktemkin.github.io/JD-sample-libraries/stdio_8h.html">The documentation for <code>uart/stdio.h</code>, or the samples below.</a>

14 Example
15 -------------
16 
17 A simple TWI interface using <a href="http://dangerousprototypes.com/bus-pirate-manual/i2c-guide/">Bus Pirate</a>-like syntax, and stdio over serial. (Compatible with Arduino boards!)
18 
19 ```C
20 #include "uart/stdio.h"
21 #include "twi/master.h"
22 
23 int main() {
24 
25  uint8_t reading_low, reading_high;
26 
27  //Set up an I2c communication at 100kHz.
28  set_up_twi_hardware(100000);
29 
30  //And take repeated light sensor readings.
31  while(1) {
32  perform_bus_pirate_twi_command("[ 0x72 0xAC [ 0x73 r s ]", &reading_low, &reading_high);
33  printf("Sensor reading: %d\n", (reading_high << 8) | reading_low);
34  _delay_ms(100);
35  }
36 
37 }
38 
39 ```
